# 色彩规范文档

## 概述

本文档定义了符合WCAG 2.1 AA级标准的色彩系统，确保所有界面元素具有良好的可访问性和对比度。

## 暗色主题色彩系统

### 背景色
- `--background-primary`: #121212 (对比度 15.8:1)
- `--background-secondary`: #1E1E1E (对比度 13.2:1)
- `--background-tertiary`: #2D2D2D (对比度 10.6:1)
- `--background-quaternary`: #3D3D3D (对比度 8.1:1)

### 文本色
- `--text-primary`: #FFFFFF (对比度 21:1)
- `--text-secondary`: #B0B0B0 (对比度 7.5:1)
- `--text-tertiary`: #808080 (对比度 5.3:1)
- `--text-disabled`: #606060 (对比度 4.2:1)

### 主色调（暗色主题专用）
- `--primary-color`: #FFD700 (对比度 15.8:1)
- `--primary-dark`: #FFC107 (对比度 13.2:1)
- `--primary-light`: #FFE55C (对比度 18.9:1)

## 高对比度主题色彩系统

专为视力障碍用户设计，使用最大对比度：
- 主色调：蓝色系（对比度 6.2:1 - 16.8:1）
- 背景色：纯白色系
- 文本色：纯黑色系
- 边框：纯黑色

## 交互元素状态规范

### 按钮状态
- **正常状态**: 使用主色调，对比度≥7.5:1
- **悬停状态**: 使用深色变体，对比度≥11.3:1
- **激活状态**: 轻微下移，添加阴影反馈
- **禁用状态**: 降低对比度至4.6:1，不可点击
- **焦点状态**: 2px轮廓线，偏移2px

### 链接状态
- **正常状态**: 下划线，对比度≥7.5:1
- **悬停状态**: 加粗下划线，对比度≥11.3:1
- **焦点状态**: 轮廓线高亮

### 输入框状态
- **正常状态**: 2px边框，对比度≥4.7:1
- **焦点状态**: 主色调边框，3px阴影
- **禁用状态**: 浅色背景，对比度≥4.6:1

## 实现代码示例

### CSS变量定义
```css
:root {
  /* 主色调系统 */
  --primary-color: #B8860B;
  --primary-dark: #8B6508;
  --primary-light: #DAA520;
  
  /* 文本系统 */
  --text-primary: #212529;
  --text-secondary: #495057;
  
  /* 背景系统 */
  --background-primary: #FFFFFF;
  --background-secondary: #F8F9FA;
}
```

### 按钮实现
```css
.btn-primary {
  background-color: var(--primary-color);
  color: var(--background-primary);
  border: 2px solid var(--primary-color);
  min-height: 44px;
}

.btn-primary:hover {
  background-color: var(--primary-dark);
  border-color: var(--primary-dark);
}
```

## 测试要求

### 对比度测试
- 所有正文文本对比度≥4.5:1
- 所有标题文本对比度≥3:1
- 所有交互元素对比度≥4.5:1

### 浏览器兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 屏幕适配
- 移动端：≥320px宽度
- 平板端：≥768px宽度
- 桌面端：≥1200px宽度

## 维护指南

1. **新增颜色**: 必须通过对比度测试
2. **修改颜色**: 需要重新测试相关组件
3. **主题切换**: 确保所有主题状态一致
4. **可访问性**: 定期使用工具检测对比度

## 工具推荐

- **对比度检查**: WebAIM Contrast Checker
- **色彩模拟**: Chrome DevTools Color Vision Deficiency
- **自动化测试**: axe-core accessibility testing
